# Octopus Energy — Customer Tariff Switch Analyser

A Streamlit web app that simulates how Octopus Energy might identify which customers would benefit from switching to a variable tariff — and which are already on the right deal.

**[Live demo →](https://octopus-tariff-switch-analyser.streamlit.app)**

---

## What this is

This tool analyses a synthetic cohort of UK energy customers, segments them by usage behaviour, and recommends whether each customer should switch from a Standard tariff to either Agile or Tracker — along with an estimated annual saving.

It is built as a portfolio project demonstrating the kind of segmentation analysis a Marketing Data Analyst at Octopus Energy would run ahead of a tariff migration campaign.

---

## What data is real vs synthetic

### Real — fetched live from the Octopus public API

Agile Octopus half-hourly unit rates (p/kWh inc. VAT) are pulled from:

```
https://api.octopus.energy/v1/products/AGILE-24-10-01/electricity-tariffs/E-1R-AGILE-24-10-01-{region}/standard-unit-rates/
```

No API key is required. The app fetches prices for the selected region and date range on load, caching results for one hour. From the real price data, three inputs are derived for the cost model:

- **agile_peak_p** — average Agile rate during 16:00–19:00 (p/kWh)
- **agile_offpk_p** — average Agile rate outside 16:00–19:00 (p/kWh)
- **agile_avg_p** — overall average across all half-hour slots (p/kWh)

These three values change when you switch region in the sidebar, which is why the tariff recommendations shift by geography.

### Synthetic — author-derived, not from Octopus

Everything else in the app is generated by the author to produce plausible results. None of the following comes from Octopus's internal data or published documentation:

**Customer segments and population weights**

Four archetypes are defined with estimated UK population weights:

| Segment | Weight | Rationale |
|---|---|---|
| Low & Stable | 25% | Retirees, single occupants, low consumption households |
| High & Stable | 30% | Families, always-home workers, high but predictable consumption |
| Peak Heavy | 28% | 9-to-5 commuters whose usage spikes at 4–7pm |
| Off-Peak Opportunist | 17% | EV owners, home workers who actively time-shift loads |

**Consumption model: base, pm, and om**

Each customer's electricity use is modelled per half-hour slot using three values:

- `base` — kWh per slot before any multipliers. Set so annual consumption lands in a realistic 2,500–4,500 kWh/yr range for all segments.
- `pm` (peak multiplier) — scales `base` during the 6 peak slots (16:00–19:00). A value of 3.50 means the customer uses 3.5× their base rate during those hours.
- `om` (off-peak multiplier) — scales `base` across the remaining 42 slots per day.

Annual kWh = (6 × base × pm + 42 × base × om) × 365

Critically, the Off-Peak Opportunist has a **lower base** than other segments (0.18 vs 0.28–0.45). The design intent is that they are not high consumers — they are smart schedulers. Their `om` of 1.10 reflects modest load-shifting (e.g. running a dishwasher overnight) rather than higher total usage. This ensures their annual kWh is comparable to other segments and the cost chart reflects tariff fit, not volume.

**Tariff cost formulas**

| Tariff | Formula | Source of numbers |
|---|---|---|
| Standard | `annual_kwh × U(24, 26) / 100` | UK Oct 2024 price cap level — publicly available from Ofgem |
| Agile (blended) | `annual_kwh × (peak_frac × agile_peak_p + (1 − peak_frac) × agile_offpk_p) / 100` | Uses real API prices; peak_frac from synthetic usage model |
| Tracker | `annual_kwh × agile_avg_p × U(1.15, 1.25) / 100` | agile_avg_p is real; the 15–25% margin is author-estimated |

The Tracker margin (15–25% above wholesale average) represents the supplier's hedging and admin cost for offering a daily variable rate. This figure is not from Octopus's pricing documentation — it is a reasonable estimate based on general knowledge of retail energy pricing. In practice Octopus's Tracker margin varies and is published on their website.

**Volatility exposure score**

A score between 0 and 1 representing how much of a customer's usage overlaps with high-price Agile windows. Drawn from a uniform distribution within segment-specific ranges:

| Segment | Range |
|---|---|
| Low & Stable | 0.30 – 0.50 |
| High & Stable | 0.40 – 0.60 |
| Peak Heavy | 0.65 – 0.90 |
| Off-Peak Opportunist | 0.10 – 0.35 |

These ranges are author-defined to reflect the intuition that Peak Heavy customers have the most exposure to volatile price windows, and Opportunists the least. They are not derived from meter data.

## The three tariffs explained

**Standard** — a fixed unit rate (~24–26p/kWh) regardless of when you use power. Predictable bills but no opportunity to benefit from cheaper off-peak electricity.

**Agile Octopus** — half-hourly prices that follow the wholesale electricity market. Cheap overnight (sometimes negative), expensive during the 4–7pm evening peak. Rewards customers who can shift flexible loads — EV charging, dishwashers, washing machines — away from peak hours.

**Tracker** — a daily rate that tracks the wholesale market plus a supplier margin. Less volatile than Agile — changes once per day rather than every 30 minutes. A middle ground for customers who want some wholesale benefit without actively managing when they use power.

```bash
git clone https://github.com/rafeeahmed1999-beep/octopus-tariff-switch-analyser.git

pip install -r requirements.txt
streamlit run otsa_app.py
```

## Deploying to Streamlit Community Cloud

1. Push this repo to GitHub (must be public)
2. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub
3. Click **Create app** → select this repo → set main file path to `otsa_app.py`
4. Click **Deploy**
